# Control Center — The Executive Nerve Center

## The Problem

SupplyChain+ has 6 modules (Connect, Engage, Educate, Intelligence, Governance, Remediation), 5 databases, 9 background jobs, and 15 AI tools. A compliance officer opening the app for the first time needs to see **one screen** that answers: "What needs my attention right now?"

The Control Center dashboard is that screen. It composes data from every module into a scannable layout without requiring navigation.

## The Dashboard Layout

The dashboard renders top-to-bottom in priority order — the most actionable information appears first:

```
┌──────────────────────────────────────────────────────┐
│  Pipeline Freshness Bar                              │  ← When was data last updated?
├──────────────────────────────────────────────────────┤
│  AI Briefing Bar                                     │  ← Daily intelligence summary
├──────────┬──────────┬──────────┬─────────────────────┤
│ High-Risk│  Urgent  │ Supplier │    Training         │  ← KPI metric cards
│ Suppliers│  Cases   │  Trends  │    Coverage         │
├──────────┴──────────┴──────────┴─────────────────────┤
│  ML Insight Cards                                    │  ← Clusters, forecasts, anomalies
├──────────────────────────────┬────────────────────────┤
│  Needs Attention Tabs (2/3)  │  AI Copilot Feed (1/3)│  ← Action center + activity feed
├──────────────────────────────┴────────────────────────┤
│  ▼ Visualizations (collapsible)                      │  ← Map, network graph, histogram
│    Risk Map │ Supply Chain │ Risk Distribution        │
└──────────────────────────────────────────────────────┘
```

### Why This Order?

1. **Freshness first.** If data is stale (jobs haven't run in 24h), nothing else matters — the officer needs to know they're looking at old information.
2. **Briefing second.** The AI-generated daily summary provides context before diving into numbers.
3. **KPI cards third.** Four numbers that capture the portfolio health at a glance.
4. **ML insights fourth.** What the background jobs discovered — patterns humans would miss.
5. **Attention tabs + copilot fifth.** The actionable queue — what to do next.
6. **Visualizations last (collapsible).** Geographic and structural views for exploration, not triage.

## Key Components

### Pipeline Freshness Bar

Shows when each of the 9 background jobs last completed. Fetches from `/api/freshness` which queries `jobRuns` for the latest `completed` run per job type.

**Why this matters:** If `case-clustering` last ran 3 days ago, the cluster insights on the dashboard are 3 days stale. The freshness bar makes this visible so officers don't act on outdated intelligence.

### AI Briefing Bar

A banner at the top showing the latest intelligence briefing. Generated by the `generate-briefing` job, which reads all intelligence tables and produces an executive summary. Fetched from `/api/metrics/briefing`.

### Metric Cards (4 KPIs)

Four cards providing portfolio-level health:

| Card | Source | What It Shows | Urgency Signal |
|------|--------|--------------|----------------|
| **High-Risk Suppliers** | `/api/metrics` | Count of suppliers with risk ≥ 75 | Red "urgent" variant if > 0 |
| **Urgent Cases** | `/api/metrics` | Count of active grievance cases | Red if > 10 |
| **Supplier Trends** | `/api/metrics` | Net improving vs. worsening (e.g., "+5") | Arrow up/down |
| **Training Coverage** | `/api/metrics` | Average completion % across all suppliers | Progress bar |

The `/api/metrics` endpoint hits all 5 databases in parallel to compute these numbers in a single API call.

### ML Insight Cards

Rendered by `MLInsightCards`, these show what background ML jobs discovered:
- Critical case clusters with supplier counts
- Rising risk forecast warnings
- Unresolved payslip anomalies by severity
- Global sentiment shift direction

Data comes from `/api/ml-insights` (see [15-ai-copilot.md](15-ai-copilot.md) for the aggregation logic).

### Needs Attention Tabs + AI Copilot

The main action center, split 2/3 + 1/3:

- **Left (2/3):** `NeedsAttentionTabs` — 4-tab attention queue (see [16-alerts-system.md](16-alerts-system.md))
- **Right (1/3):** `AICopilotFeed` — Dynamic insight + activity stream (see [15-ai-copilot.md](15-ai-copilot.md))

### Visualizations (Collapsible)

Three heavy visualization components behind a collapsible toggle:

1. **Geographic Risk Map** — D3-geo + React Simple Maps rendering supplier locations colored by risk level. Uses factory coordinates from SQL Server `CompanyPost` table.
2. **Supply Chain Network** — React Flow / Xyflow graph showing brand → supplier hierarchy.
3. **Risk Distribution Chart** — Recharts histogram showing the distribution of risk scores across all suppliers (how many at 0-20, 20-40, etc.).

## Lazy Loading Pattern

All three visualization components use Next.js `dynamic()` imports:

```typescript
const GeographicRiskMap = dynamic(
  () => import("@/components/dashboard/geographic-risk-map")
    .then((m) => ({ default: m.GeographicRiskMap })),
  { ssr: false, loading: () => <div className="h-[300px] bg-muted animate-pulse" /> }
);
```

**Why?**
- These components pull in D3, React Flow, and chart libraries — large bundles that would slow initial page load.
- `ssr: false` prevents server-side rendering of browser-dependent libraries (D3 needs `window`, React Flow needs `requestAnimationFrame`).
- Loading skeletons prevent layout shift while the components load.

The collapsible wrapper (`Collapsible` from shadcn/ui) lets users hide visualizations entirely, focusing on the actionable top half.

## View Context Filtering

The dashboard respects the three-level scope system (see [13-frontend-and-data-flow.md](13-frontend-and-data-flow.md)):

| Level | What Changes |
|-------|-------------|
| **Portfolio** (default) | Shows all suppliers across all brands |
| **Brand** | Filters to one parent company's suppliers (`parentCompanyId` param) |
| **Supplier** | N/A — navigates to supplier detail page instead |

When a brand is selected, the dashboard header changes to "Viewing suppliers for {brand name}" and all metric/insight queries include the `parentCompanyId` filter. This happens via the `useView()` hook from `ViewContext`.

## Data Fetching Strategy

The dashboard makes 4-5 parallel queries on mount:

```typescript
const { data: metrics }    = useQuery({ queryKey: ["metrics", parentCompanyId], ... });
const { data: suppliersRes } = useQuery({ queryKey: ["suppliers", parentCompanyId], ... });
const { data: brands }     = useQuery({ queryKey: ["brands"], ... });
// Plus: ml-insights, alerts, briefing, activities (fetched by child components)
```

Each child component manages its own data fetching via React Query. This means:
- **No waterfall:** All components start fetching simultaneously on render.
- **Independent loading:** The metric cards can show data while the map is still loading.
- **Independent refetch:** Alerts auto-refresh every 60s without re-fetching metrics.

React Query handles caching, deduplication (multiple components requesting the same data get one fetch), and background refetching.
